## 一、技术架构设计

### 1.1 前端技术栈选型对比分析

**选定方案: Taro 4.x + React + TypeScript**

#### 技术方案对比矩阵

| 维度 | **Taro 4.x + React** | UniApp + Vue3 | 原生微信小程序 |
|------|---------------------|--------------|----------------|
| **开发效率** | ⭐⭐⭐⭐⭐ 一套代码多端运行,React生态成熟 | ⭐⭐⭐⭐⭐ 同样支持多端 | ⭐⭐ 仅支持微信平台 |
| **学习成本** | ⭐⭐⭐⭐ React Hooks需适应,但生态文档丰富 | ⭐⭐⭐⭐ Vue生态成熟,上手快 | ⭐⭐⭐⭐⭐ 官方文档完善 |
| **性能表现** | ⭐⭐⭐⭐⭐ Taro 4.x编译优化大幅提升,接近原生 | ⭐⭐⭐⭐ 编译时优化,接近原生 | ⭐⭐⭐⭐⭐ 最优 |
| **跨平台能力** | ⭐⭐⭐⭐⭐ 支持微信/支付宝/百度/H5/React Native | ⭐⭐⭐⭐⭐ 支持微信/支付宝/百度/H5/App | ⭐ 仅微信 |
| **生态支持** | ⭐⭐⭐⭐⭐ React组件库丰富(NutUI/Taro-UI),社区活跃 | ⭐⭐⭐⭐⭐ uView/uni-ui等成熟UI库 | ⭐⭐⭐ 官方组件有限 |
| **维护成本** | ⭐⭐⭐⭐ 统一代码库,TypeScript类型安全 | ⭐⭐⭐⭐ 统一代码库,易维护 | ⭐⭐⭐⭐ 单平台简单 |
| **AI集成友好度** | ⭐⭐⭐⭐⭐ 支持npm包,React生态SDK丰富 | ⭐⭐⭐⭐ 支持npm包,易于集成SDK | ⭐⭐⭐ 受限于微信环境 |
| **适合独立开发** | ⭐⭐⭐⭐⭐ Taro 4.x性能优化显著,开发效率高,快速验证MVP | ⭐⭐⭐⭐⭐ 开发效率高,快速验证MVP | ⭐⭐ 跨平台扩展困难 |

**选型结论**: 采用 **Taro 4.x + React + TypeScript** 作为前端技术栈,理由:
- Taro 4.x版本性能大幅提升,编译优化更智能,运行时性能接近原生小程序
- 独立开发者需要快速验证MVP,跨平台能力为后续扩展预留空间
- React Hooks + TypeScript提供强大的类型安全和组件化开发体验
- Taro 4.x支持React 18,可利用并发特性优化复杂交互(数值系统、答题流程)
- 丰富的UI组件库(NutUI React/Taroify)可快速搭建界面
- React生态成熟,AI相关SDK和工具链支持更完善

### 1.2 后端服务架构设计

**选定方案: 传统服务器架构(NestJS + MySQL)**

#### 架构方案对比

| 维度 | **传统服务器(NestJS)** | Serverless(函数计算) | 微服务架构 |
|------|---------------------|------------------------|------------|
| **初期成本** | ⭐⭐⭐⭐ 固定成本,可控 | ⭐⭐⭐⭐⭐ 按量付费,0流量0成本 | ⭐ 多服务器成本高 |
| **运维复杂度** | ⭐⭐⭐ 需监控部署 | ⭐⭐⭐⭐⭐ 无需管理服务器 | ⭐⭐ 需K8s等服务网格 |
| **弹性伸缩** | ⭐⭐⭐ 需手动配置 | ⭐⭐⭐⭐⭐ 自动扩容 | ⭐⭐⭐⭐ 自动但复杂 |
| **冷启动延迟** | ⭐⭐⭐⭐⭐ 持续运行 | ⭐⭐⭐ 首次调用有延迟 | ⭐⭐⭐⭐⭐ |
| **适合MVP** | ⭐⭐⭐⭐ 传统方案可靠,成本可控 | ⭐⭐⭐⭐⭐ 快速上线,低成本验证 | ⭐ 过度设计 |
| **长期成本** | ⭐⭐⭐⭐ 可优化,批量生成题目更经济 | ⭐⭐⭐ 量大时成本上升 | ⭐⭐⭐ 复杂度高 |

**架构设计**:

```
前端(Taro 4.x) 
  ↓ HTTPS
Nginx(反向代理、静态资源服务)
  ↓
传统服务器(NestJS)
  ├── /api/game/start        # 开始游戏
  ├── /api/game/next         # 下一题
  ├── /api/game/submit       # 提交答案
  ├── /api/report/generate   # 生成报告
  └── /api/stats/parallel    # 平行宇宙数据
  ↓
MySQL数据库
  ├── game_sessions表       # 游戏会话
  ├── game_records表        # 答题记录
  ├── stats_cache表         # 统计数据缓存
  └── presets表             # 预设词条配置
  ↓
本地文件系统
  ├── /questions/           # 预生成题库JSON
  ├── /images/              # 预生成场景图片
  └── /reports/             # 生成的报告缓存
```

**为什么选传统服务器**:
- 题目采用一次性AI生成后存储到静态库,重复使用,大幅降低API调用成本
- 传统服务器固定成本可控(¥200-300/月),适合批量生成场景
- 独立开发者可完全掌控服务器,便于优化和管理
- 本地存储题库和图片,避免重复调用AI服务,月AI成本<¥50

### 1.3 AI服务集成方案

**选定方案: 阿里云通义千问(qwen-turbo / qwen-plus)**

#### 大模型API对比

| 模型 | 中文理解 | 成本(百万tokens) | 响应速度 | 上下文长度 | 适合场景 |
|------|---------|-----------------|---------|-----------|---------|
| **通义千问-turbo** | ⭐⭐⭐⭐⭐ | ¥1-2 | ⭐⭐⭐⭐⭐ | 32K | **题目生成、即时反馈** |
| **通义千问-plus** | ⭐⭐⭐⭐⭐ | ¥4-8 | ⭐⭐⭐⭐ | 32K | 复杂报告生成 |
| OpenAI GPT-4 | ⭐⭐⭐⭐ | ¥60-120 | ⭐⭐⭐ | 128K | 英文场景更优 |
| 豆包/文心一言 | ⭐⭐⭐⭐ | ¥2-5 | ⭐⭐⭐⭐ | 8-32K | 备选方案 |

**AI服务分层调用策略**:

```
低延迟场景(用户交互):
  - 题目生成: qwen-turbo (成本¥0.01/次, 响应<2s)
  - 即时反馈: qwen-turbo (成本¥0.005/次)
  - 数值计算: 前端规则引擎(不调用AI)

高复杂度场景(离线/异步):
  - 复盘报告: qwen-plus (成本¥0.05/次)
  - 平行宇宙数据聚合: 数据库查询(不调用AI)
```

**成本控制方案**:
1. **一次性生成策略**: 每种预设组合+阶段的题目一次性AI生成,存储到本地静态库
2. **图片预生成**: 场景图片一次性生成后存储到服务器,重复使用
3. **题库版本管理**: 定期更新题库,但保持版本兼容
4. **按需调用AI**: 仅报告生成时调用AI,日常答题使用预生成题库
5. **成本预估**: 月成本<¥350(服务器¥200-300 + AI调用<¥50)

### 1.4 数据库设计方案

**选定方案: MySQL(RDS) + Redis缓存**

#### 核心表结构设计

```sql
-- 预设词条表
CREATE TABLE presets (
  id INT PRIMARY KEY AUTO_INCREMENT,
  category VARCHAR(50) NOT NULL,  -- 'npd_mbti', 'bipolar_mbti'
  primary_tag VARCHAR(20),         -- 'NPD', '双向'
  secondary_tag VARCHAR(20),       -- 'INTJ', 'ENFP'
  knowledge_json JSON,             -- 知识点关键词
  status TINYINT DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 游戏会话表
CREATE TABLE game_sessions (
  session_id VARCHAR(64) PRIMARY KEY,
  user_id VARCHAR(64),             -- MVP可为空(游客模式)
  preset_id INT,
  current_stage VARCHAR(20),       -- 'meet', 'love', 'breakup', 'marriage'
  current_score INT DEFAULT 100,   -- 情绪能量/自尊值
  status ENUM('playing', 'passed', 'failed', 'abandoned'),
  stage_history JSON,              -- 各阶段得分记录
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  finished_at TIMESTAMP NULL
);

-- 答题记录表
CREATE TABLE game_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  session_id VARCHAR(64),
  question_index INT,
  question_text TEXT,
  options_json JSON,               -- 选项列表
  user_choice INT,                 -- 用户选择
  is_correct BOOLEAN,
  score_change INT,                -- 分数变化
  ai_feedback TEXT,                -- AI即时讲解
  stage VARCHAR(20),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_session (session_id)
);

-- 统计数据缓存表(平行宇宙功能)
CREATE TABLE stats_cache (
  id INT PRIMARY KEY AUTO_INCREMENT,
  preset_id INT,
  stage VARCHAR(20),
  question_index INT,
  total_answers INT DEFAULT 0,     -- 总答题数
  correct_count INT DEFAULT 0,     -- 正确数
  choice_distribution JSON,        -- 各选项选择次数
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_preset_stage_q (preset_id, stage, question_index)
);
```

**设计原则**:
- 无需用户系统,采用匿名会话模式,降低门槛
- JSON字段存储复杂结构(选项、历史记录),避免过度设计
- 统计数据采用缓存表,定时聚合,避免实时计算压力
- 预设题库表存储AI一次性生成的题目,避免重复调用

### 3.2 AI题目生成算法和调用机制

**核心Prompt模板设计**:

```typescript
// Prompt模板(后端函数中使用)
const QUESTION_GENERATION_PROMPT = `
你是一个专业的心理学剧本设计师,擅长创建沉浸式人际关系模拟场景。

## 任务
为用户生成关于【{{primary_tag}} + {{secondary_tag}}】的分阶段闯关题目。

## 当前阶段
{{current_stage}} (相识/热恋/冲突/分手或结婚)

## 要求
1. 生成3道题目,每道题包含:
   - 情境描述(100字以内,贴近真实场景)
   - 4个选项(一个最优解,一个危险红线,两个中等选项)
   - 每题标注正确选项索引(0-3)
   - 每题标注分数变化(-30到+20)
   - 即时反馈文案(50字以内,解释为何对错)

2. 难度递增规则:
   - 相识阶段: 简单,明显陷阱
   - 热恋阶段: 中等,隐藏操控手段
   - 冲突阶段: 困难,心理博弈
   - 分手/结婚: 极难,关键决策

3. 红线规则: 某些选项直接判定失败(如"容忍暴力")

## 输出格式(JSON)
{
  "stage": "{{current_stage}}",
  "questions": [
    {
      "id": 1,
      "scene": "场景描述...",
      "image_prompt": "动漫风格,情侣在咖啡厅对话,暖色调",
      "options": ["选项A", "选项B", "选项C", "选项D"],
      "correct_index": 2,
      "score_change": 15,
      "is_redline": false,
      "feedback": "心理学解释..."
    }
  ]
}

请严格按照JSON格式输出,不要包含其他文字。
`;
```

**AI调用函数实现**:
```typescript
// 阿里云函数计算 - generateQuestions.ts
import { openai } from '@alicloud/ai-gateway';

export async function handler(event: any) {
  const { session_id, stage } = JSON.parse(event);
  
  // 从数据库获取会话信息
  const session = await getSession(session_id);
  const preset = await getPreset(session.preset_id);
  
  // 构建Prompt
  const prompt = QUESTION_GENERATION_PROMPT
    .replace('{{primary_tag}}', preset.primary_tag)
    .replace('{{secondary_tag}}', preset.secondary_tag)
    .replace('{{current_stage}}', stage);
  
  // 调用通义千问
  const response = await openai.chat.completions.create({
    model: 'qwen-turbo',
    messages: [{ role: 'user', content: prompt }],
    temperature: 0.8,
    max_tokens: 2000,
    response_format: { type: 'json_object' }
  });
  
  const questions = JSON.parse(response.choices[0].message.content);
  
  // 缓存到Redis(避免重复生成)
  await cacheQuestions(session_id, stage, questions);
  
  return {
    statusCode: 200,
    body: JSON.stringify({ session_id, stage, questions })
  };
}
```

**关键设计决策**:
1. **流式输出**: 使用SSE返回AI响应,用户可实时看到题目生成进度
2. **JSON格式约束**: 强制AI输出结构化数据,避免解析错误
3. **缓存机制**: 相同阶段相同预设的题目缓存24小时,降低成本
4. **降级策略**: AI超时或失败时,从预设题库随机抽取

### 3.3 用户答题交互逻辑

**状态管理设计** (使用React Zustand + TypeScript):
```typescript
// store/game.ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

interface GameState {
  session_id: string;
  current_stage: 'meet' | 'love' | 'conflict' | 'ending';
  current_question_index: number;
  questions: Question[];
  score: number;           // 当前分数(0-100)
  score_history: number[]; // 分数变化历史
  answers: AnswerRecord[]; // 答题记录
  status: 'playing' | 'feedback' | 'passed' | 'failed';
  submitAnswer: (questionIndex: number, choiceIndex: number) => Promise<void>;
  advanceStage: () => Promise<void>;
  endGame: (result: 'passed' | 'failed') => Promise<void>;
}

export const useGameStore = create<GameState>()(
  persist(
    (set, get) => ({
      session_id: '',
      current_stage: 'meet',
      current_question_index: 0,
      questions: [],
      score: 100,
      score_history: [100],
      answers: [],
      status: 'playing',
      
      submitAnswer: async (questionIndex: number, choiceIndex: number) => {
        const state = get();
        const question = state.questions[questionIndex];
        const isCorrect = choiceIndex === question.correct_index;
        const isRedline = question.is_redline && !isCorrect;
        
        const scoreChange = isCorrect ? question.score_change : -Math.abs(question.score_change);
        const newScore = Math.max(0, Math.min(100, state.score + scoreChange));
        
        const newAnswer = {
          question_index: questionIndex,
          choice: choiceIndex,
          is_correct: isCorrect,
          score_change: scoreChange
        };
        
        // 判断游戏状态
        if (newScore <= 0 || isRedline) {
          set({
            score: newScore,
            score_history: [...state.score_history, newScore],
            answers: [...state.answers, newAnswer],
            status: 'failed'
          });
          await get().endGame('failed');
        } else if (questionIndex === state.questions.length - 1) {
          set({
            score: newScore,
            score_history: [...state.score_history, newScore],
            answers: [...state.answers, newAnswer]
          });
          await get().advanceStage();
        } else {
          set({
            score: newScore,
            score_history: [...state.score_history, newScore],
            answers: [...state.answers, newAnswer],
            current_question_index: questionIndex + 1,
            status: 'playing'
          });
        }
        
        // 提交到后端
        await submitAnswerAPI({
          session_id: state.session_id,
          question_index: questionIndex,
          choice: choiceIndex,
          is_correct: isCorrect,
          score_change: scoreChange
        });
      },
      
      advanceStage: async () => {
        const state = get();
        const stageOrder = ['meet', 'love', 'conflict', 'ending'] as const;
        const currentIndex = stageOrder.indexOf(state.current_stage);
        
        if (currentIndex < stageOrder.length - 1) {
          const nextStage = stageOrder[currentIndex + 1];
          
          // 请求下一阶段题目
          const nextQuestions = await fetchNextStageAPI({
            session_id: state.session_id,
            stage: nextStage
          });
          
          set({
            current_stage: nextStage,
            current_question_index: 0,
            questions: nextQuestions,
            status: 'playing'
          });
        } else {
          set({ status: 'passed' });
          await get().endGame('passed');
        }
      },
      
      endGame: async (result: 'passed' | 'failed') => {
        const state = get();
        set({ status: result });
        
        await endGameAPI({
          session_id: state.session_id,
          result,
          final_score: state.score,
          answers: state.answers
        });
      }
    }),
    {
      name: 'game-storage',
      partialize: (state) => ({
        session_id: state.session_id,
        score: state.score,
        status: state.status
      })
    }
  )
);

// utils/scoreCalculator.ts
interface ScoreRule {
  is_correct: boolean;
  is_redline: boolean;
  base_change: number;  // 基础分数变化
  stage: string;
}

export function calculateScoreChange(rule: ScoreRule): number {
  let change = rule.base_change;
  
  // 红线选项直接扣30分
  if (rule.is_redline && !rule.is_correct) {
    change = -30;
  }
  
  // 阶段难度系数
  const stageMultiplier = {
    'meet': 1.0,      // 相识阶段
    'love': 1.2,      // 热恋阶段
    'conflict': 1.5,  // 冲突阶段
    'ending': 2.0     // 结局阶段
  };
  
  change *= stageMultiplier[rule.stage] || 1.0;
  
  // 连续正确奖励
  // (在游戏状态中维护连续正确次数)
  
  return Math.round(change);
}

export function checkGameOver(currentScore: number): boolean {
  return currentScore <= 0;
}

export function calculateAccuracy(answers: AnswerRecord[]): number {
  const correct = answers.filter(a => a.is_correct).length;
  return Math.round((correct / answers.length) * 100);
}

FROM game_sessions
GROUP BY stage;

-- 平均答题正确率
SELECT 
  AVG(correct_count / total_count * 100) as avg_accuracy
FROM (
  SELECT 
    session_id,
    SUM(CASE WHEN is_correct THEN 1 ELSE 0 END) as correct_count,
    COUNT(*) as total_count
  FROM game_records
  GROUP BY session_id
) t;
```

**状态机实现**:
```typescript
// 闯关状态机
class GameStageMachine {
  private currentStage: GameStage;
  private score: number;
  
  constructor() {
    this.currentStage = 'meet';
    this.score = 100;
  }
  
  // 判断是否可进入下一阶段
  canAdvance(): boolean {
    const config = STAGE_CONFIG[this.currentStage];
    return this.score >= config.pass_threshold;
  }
  
  // 进入下一阶段
  advance(): GameStage | 'game_over' {
    if (!this.canAdvance()) {
      return 'game_over';
    }
    
    const nextStage = STAGE_CONFIG[this.currentStage].next;
    if (!nextStage) {
      return 'completed';
    }
    
    this.currentStage = nextStage;
    return nextStage;
  }
  
  // 红线直接结束
  triggerRedline(): 'game_over' {
    this.score = 0;
    return 'game_over';
  }
}
```

**与AI生成的联动**:
- 每次阶段切换时,调用AI生成新阶段题目
- AI根据`stage`参数调整难度和场景
- 失败时可选择"复活"(后期商业化功能)

### 5.3 数值可视化系统前端实现

**技术选型**: 
- **方案A**: 纯CSS动画 (轻量,适合简单进度条) ✅ MVP推荐
- **方案B**: ECharts for Taro (功能强大,支持复杂图表)
- **方案C**: Canvas自绘 (性能最优,但开发成本高)

**MVP实现方案** (CSS + React Hooks):
```tsx
import React, { useState, useEffect, useMemo } from 'react';
import { View, Text } from '@tarojs/components';
import './index.scss';

const ScoreVisual: React.FC<{ score: number }> = ({ score }) => {
  const [showChange, setShowChange] = useState(false);
  const [changeAmount, setChangeAmount] = useState(0);
  const [prevScore, setPrevScore] = useState(score);

  const scoreGradient = useMemo(() => {
    if (score > 70) return 'linear-gradient(90deg, #56AB2F, #A8E063)';
    if (score > 40) return 'linear-gradient(90deg, #F7971E, #FFD200)';
    return 'linear-gradient(90deg, #FF416C, #FF4B2B)';
  }, [score]);

  const changeType = useMemo(() => 
    changeAmount > 0 ? 'positive' : 'negative',
    [changeAmount]
  );

  // 监听分数变化,触发显示动画
  useEffect(() => {
    const diff = score - prevScore;
    if (diff !== 0) {
      setChangeAmount(diff);
      setShowChange(true);
      setPrevScore(score);
      const timer = setTimeout(() => setShowChange(false), 1000);
      return () => clearTimeout(timer);
    }
  }, [score, prevScore]);

  return (
    <View className="score-visual">
      {/* 主数值条 */}
      <View className="main-bar">
        <View className="bar-track">
          <View 
            className="bar-fill"
            style={{ 
              width: `${score}%`,
              background: scoreGradient,
              transition: 'width 0.5s ease'
            }}
          ></View>
        </View>
        <Text className="score-text">{score}</Text>
      </View>
      
      {/* 分数变化浮层 */}
      {showChange && (
        <View className={`change-indicator ${changeType}`}>
          {changeAmount > 0 ? '+' : ''}{changeAmount}
        </View>
      )}
    </View>
  );
};

export default ScoreVisual;
```
